# AI Workbench

这是一个 AI 工具集仓库，包含多 Agent 流水线团队、独立角色、可复用技能和通用规则。

> **本文件由 Claude Code 自动加载。** Copilot CLI 用户请参考 `.github/copilot-instructions.md`。

---

## 仓库结构

```
teams/          多Agent协作流水线（主编调度 + 多角色并行）
agents/         独立单角色提示词（不属于任何团队，可复用）
skills/         可复用技能（一个 SKILL.md 专注一件事）
rules/          通用规则（跨团队复用）
tools/          工具配置和脚本（MCP模板、辅助脚本）
```

## 可用团队

### wechat-xhs-publisher — 微信小红书发布助手

5人流水线团队，实现从选题到发布的全流程内容运营。

| 角色 | 职责 |
|------|------|
| 主编 | 选题策划、审核、调度、质检 |
| 撰稿师 | 微信HTML文章 + 小红书文案 |
| 视觉师 | 智能配图分析 + AI封面/配图生成 |
| 微运营 | 微信公众号草稿+群发 |
| 红运营 | 小红书笔记发布（多图3-9张） |

**完整文档**：`teams/wechat-xhs-publisher/team.md`

**使用方式**（Claude Code 单Agent模式）：

由于 Claude Code 不支持多 Agent 并行，你需要依次切换角色完成各阶段工作。在每个阶段开头声明当前角色：

```
【当前角色：主编】确认选题...
【当前角色：撰稿师】撰写文案...
【当前角色：视觉师】智能配图分析...
【当前角色：微运营】创建微信草稿...
【当前角色：红运营】发布小红书笔记...
```

**SOP 路由表**：

| 用户意图 | 流程 |
|---------|------|
| "写篇文章发微信和小红书" | Workflow 1（双平台完整流程） |
| "写篇文章只发微信/小红书" | Workflow 1.5（单平台） |
| "帮我改写这段内容" | Workflow 2（改写，可衔接到发布） |
| "已有文案，帮我发" | Workflow 3（直接发布） |
| "读一下这个项目/视频/链接，写篇文章发出去" | Workflow 4（素材驱动） |

**⚠️ 铁律**：
1. 必须等用户确认"发布"才能执行发布，严禁跳过确认闸门
2. 微信 HTML 必须全内联样式，所见即所得
3. 图片必须：清晰可读 + 对比鲜明 + 浅色主题优先
4. 每个阶段完成后向用户简要通报

**依赖 MCP**：wechat-official-account、xiaohongshu-mcp

## 核心品牌规范

> 完整规范见 `teams/wechat-xhs-publisher/rules/brand-guidelines.md`

| 用途 | 色值 |
|------|------|
| 主强调色 | `#D97757`（暖橙色） |
| 页面背景 | `#FBF8F5`（暖白） |
| 代码块背景 | `#F5F0EB`（浅暖灰） |
| 正文文字 | `#333` / `#2C2C2A`（深棕） |

**字体**：正文用系统字体栈，代码用等宽字体栈，禁止外部字体。

**尺寸**：微信封面 900x500，正文配图 ≤600px 宽，小红书 1080x1440 (3:4)。

**图片质量**：
- 清晰可读：文字够大、线条 ≥ 2px、手机端可辨
- 对比鲜明：深色文字 + 浅色背景，禁止低对比度配色
- 浅色主题：默认浅色/白色底色，仅代码截图和金句卡片允许深色

**微信 HTML 规则**：
- 全内联样式，禁止 `<style>` 标签 / 外部 CSS / CSS 变量
- 禁止：linear-gradient、backdrop-filter、box-shadow、CSS 动画、@media 查询
- 纯色替代渐变

**小红书排版**：
- 段落 ≤ 4 行，emoji 作视觉节奏点，关键信息用【】突出
- 标题 ≤ 20 字，话题标签 3-5 个

## 合规底线

> 完整规则见 `teams/wechat-xhs-publisher/rules/xiaohongshu-compliance.md` 和 `wechat-compliance.md`

**小红书**：
- 标题提品牌必须中性语气，禁止安利（真香/神器/YYDS）
- AI 内容必须标注
- 禁止医疗/金融/投资建议

**微信**：
- 禁止诱导分享/关注
- 禁止夸大/虚假宣传
- 图文消息正文 ≤ 20000 字

## 添加新内容

- **新团队** → `teams/your-team/` 目录，包含 `team.md` + `agents/` + `rules/`
- **新角色** → `agents/role-name.md`
- **新技能** → `skills/skill-name/SKILL.md`
- **新规则** → `rules/rule-name.md`
- **新工具配置** → `tools/mcp-configs/` 或 `tools/scripts/`
