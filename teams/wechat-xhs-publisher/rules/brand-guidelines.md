# 品牌视觉与排版规范

> 本文件是团队所有成员共享的品牌规范唯一来源。所有配色、字体、尺寸规范以本文件为准。
> 最后更新：2026-06-02

## 一、品牌配色

| 用途 | 色值 | 说明 |
|------|------|------|
| 主强调色 | `#D97757` | 暖橙色，用于标题装饰、链接、按钮、分隔线、重点高亮、封面/首图主色调 |
| 页面背景 | `#FBF8F5` | 暖白，用于页面背景、首图背景、信息图底色 |
| 代码块背景 | `#F5F0EB` | 浅暖灰，用于代码块背景 |
| 正文文字 | `#333` 或 `#2C2C2A` | 深棕色，正文主色 |
| 整体风格 | 浅色主题 | 干净专业，科技感+人文感 |

> ⚠️ 每张封面/首图必须以 `#D97757` 作为视觉锚点。

## 二、图片质量强制规范

所有由视觉师生成的图片（封面、首图、正文配图、内页图）必须遵循以下质量标准：

### 2.1 清晰可读
- 文字元素（标题、标签、图注）字号不得过小，移动端阅读时必须肉眼可辨
- 线条、箭头、连接线等图形元素线宽 ≥ 2px，确保缩放后不糊
- 图表/信息图中的数据文字与标签必须清晰，不得与背景融为一体

### 2.2 内容与背景对比鲜明
- **浅色背景 + 深色文字**：正文/标签/说明文字使用深色（`#333` / `#2C2C2A`），底色使用浅色（`#FBF8F5` / `#FFFFFF`）
- **深色文字 + 强调色点缀**：关键元素用 `#D97757` 突出，但文字主体保持深色，不可全用浅色文字
- **禁止**：浅色文字 + 浅色背景、深色背景 + 深色文字、低对比度配色（如灰色文字 + 白色背景）
- 图表/架构图中的节点与连接线颜色必须有足够区分度

### 2.3 浅色主题优先
- 所有图片默认使用**浅色/白色底色**，主背景色为 `#FBF8F5`（暖白）或 `#FFFFFF`（纯白）
- 仅在以下场景允许深色底色：
  - **代码截图**：可使用深色编辑器主题（如 One Dark），但必须搭配 `#D97757` 橙色边框或标题栏
  - **金句卡片**：可使用 `#D97757` 暖橙色底色 + 白色文字，但不可用纯黑/深灰底
- 封面/首图/信息图/架构图/流程图/逻辑图等**一律使用浅色主题**

### 2.4 移动端适配
- 所有关键信息必须在 375px 宽度（手机屏幕）下仍然清晰可辨
- 测试标准：图片缩放到手机尺寸后，文字仍可阅读、图形仍可辨识

## 三、字体规范

| 用途 | 字体栈 | 说明 |
|------|--------|------|
| 正文 | `-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif` | 系统字体栈，微信/小红书通用 |
| 代码 | `"SF Mono", "Fira Code", "Consolas", monospace` | 代码字体栈 |

**禁止使用**：Google Fonts 或任何外部字体引用。

## 四、微信公众号尺寸规范

| 元素 | 尺寸 | 比例 | 说明 |
|------|------|------|------|
| 封面大图 | 900×500 px | 2.35:1 | 文章封面，主色调 #D97757 |
| 封面小图 | 200×200 px | 1:1 | 多图文封面缩略图 |
| 正文配图 | 宽度 ≤ 600px | 建议 16:9 或 4:3 | 正文内嵌图片，高度自适应 |
| 正文配图（信息图） | 宽度 ≤ 600px | 竖版可用 3:4 | 数据可视化、步骤说明等信息图 |

### 微信正文配图规范
- **配图决策由视觉师智能判断**：视觉师阅读全文后自动分析哪里需要配图、配什么类型，无需撰稿师手动标记
- 配图节奏参考：每 3-4 段文字（约 500-800 字）可考虑插入一张配图，具体由内容密度决定
- 配图类型优先级：架构图/流程图 > 信息图 > 步骤图 > 对比图 > 场景图 > 装饰图
- 配图与正文配合：配图应出现在对应文字之后，而非之前
- 配图必须带内联样式的 `<img>` 标签，设置 `max-width: 100%`
- 同一篇文章的所有配图风格统一（同色系、同画风、同圆角/阴影风格）
- **配图类型对照**：
  - 描述系统/模块关系 → 架构图
  - 描述流程/步骤 → 流程图/步骤图
  - 描述因果/逻辑 → 逻辑图
  - 描述数据/对比 → 信息图/对比图
  - 描述概念/原理 → 概念图
  - 总结观点/金句 → 金句卡片

## 五、小红书尺寸规范

| 元素 | 尺寸 | 比例 | 说明 |
|------|------|------|------|
| 首图 | 1080×1440 px | 3:4 | 笔记首图，竖版 |
| 内页图 | 1080×1440 px | 3:4 | 笔记内页图，与首图统一尺寸 |

### 小红书多图规范
- **多图规划由视觉师智能判断**：视觉师阅读笔记全文后自动分析内容结构，规划多图轮播叙事
- 图片数量：3-9 张（最少 3 张，推荐 5-7 张），具体数量由内容丰富度决定
- 所有图片尺寸统一为 1080×1440 px（3:4 竖版）
- 首图是门面：必须最吸睛，决定用户是否点开
- 图文互补而非重复：图片展示视觉信息，文字补充感受和细节
- 末图建议放金句/总结/互动引导卡片
- 每张图一个核心信息，文字精简，视觉突出
- 多图叙事顺序：首图（吸引）→ 核心要点图（2-3张）→ 细节图（1-2张）→ 总结图（1张）

## 六、HTML 渲染规则（微信公众号强制）

以下规则适用于所有输出到微信公众号的 HTML 内容：

### 禁止使用的 CSS
- ❌ `linear-gradient` / `radial-gradient`（渐变）
- ❌ `backdrop-filter` / `filter`（滤镜）
- ❌ `box-shadow`（阴影）
- ❌ CSS 动画：`@keyframes` / `animation` / `transition`
- ❌ `@media` 查询（响应式断点）
- ❌ `<style>` 标签 / `<link>` 外部样式表
- ❌ CSS 变量（`var(--xxx)`）
- ❌ Google Fonts 或任何外部字体引用

### 强制使用的写法
- ✅ **全内联样式**：所有 CSS 写在元素的 `style` 属性中
- ✅ **系统字体栈**：见上方字体规范
- ✅ **纯色替代渐变**：Hero/标题背景用 `#D97757` 纯色

### 字号规范
- 正文：15-16px
- 小标题：18-20px
- 代码：14px

### 发布铁律
> ⚠️ **所见即所得。** 产出的 HTML 是什么样，发布就是什么样，不允许"先做漂亮再降级"。

## 七、小红书正文排版规则

- 段落间空行分隔，每段不超过 3-4 行
- 使用 emoji 作为视觉节奏点（每段 1-2 个，不过度）
- 关键信息用 **【】括号标注** 或换行突出
- 正文末尾留一行引导互动（点赞/收藏/评论）
- 标题 ≤ 20 字
- 话题标签 3-5 个（1-2 个大类标签 + 2-3 个精准标签）
